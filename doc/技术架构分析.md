# EAP-Common 技术架构分析

## 项目概述

EAP-Common 是 OpenEA 企业应用平台的基础框架项目，提供依赖包、基础框架、基本工具以及技术依赖包管理。该项目采用模块化设计，为整个 EAP 生态系统提供统一的技术基础设施。

## 技术栈

### 核心技术
- **Java版本**: JDK 17
- **Spring Boot**: 3.5.6
- **Maven**: 多模块项目管理
- **项目版本**: 2.8.5

### 主要依赖技术
- **Web框架**: Spring Boot 3.x
- **数据库**: 
  - MyBatis Plus 3.5.12
  - Druid 1.2.25 (连接池)
  - 多数据源支持 (dynamic-datasource 4.3.1)
- **缓存**: Redis (Redisson 3.41.0)
- **安全**: Spring Security + JWT
- **文档**: SpringDoc 2.8.9, Knife4j 4.6.0
- **工具库**: 
  - Hutool 5.8.35
  - MapStruct 1.6.3
  - Lombok 1.18.38
  - Guava 33.4.8-jre

## 项目架构

### 模块结构

```
eap-common/
├── eap-dependencies/              # 依赖管理模块
├── eap-boot-common/              # 基础公共模块
│   ├── eap-common/               # 核心公共组件
│   └── eap-common-biz-api/       # 业务API接口
├── eap-boot-framework/           # 框架模块集合
│   ├── eap-spring-boot-starter-security/      # 安全框架
│   ├── eap-spring-boot-starter-mybatis/       # 数据库框架
│   ├── eap-spring-boot-starter-redis/         # 缓存框架
│   ├── eap-spring-boot-starter-web/           # Web框架
│   ├── eap-spring-boot-starter-protection/    # 保护机制
│   ├── eap-spring-boot-starter-test/          # 测试框架
│   ├── eap-spring-boot-starter-excel/         # Excel处理
│   ├── eap-spring-boot-starter-job/           # 任务调度
│   ├── eap-spring-boot-starter-monitor/       # 监控
│   ├── eap-spring-boot-starter-mq/            # 消息队列
│   ├── eap-spring-boot-starter-websocket/     # WebSocket
│   ├── eap-spring-boot-starter-biz-data-permission/ # 数据权限
│   ├── eap-spring-boot-starter-biz-ip/        # IP处理
│   └── eap-spring-boot-starter-biz-tenant/    # 多租户
└── eap-cloud-framework/          # 微服务框架 (可选)
    ├── eap-spring-boot-starter-env/           # 环境配置
    ├── eap-spring-boot-starter-rpc/           # RPC调用
    └── eap-spring-cloud-starter-rpc-all/      # RPC全套
```

### 核心框架模块详解

#### 1. 安全框架 (eap-spring-boot-starter-security)
- **功能**: 提供统一的安全认证和授权机制
- **特性**:
  - JWT Token 认证
  - Mock 模式支持
  - 密码加密
  - 权限控制
  - 登录用户信息管理

#### 2. 数据库框架 (eap-spring-boot-starter-mybatis)
- **功能**: 数据库访问层统一封装
- **特性**:
  - MyBatis Plus 增强
  - 自定义查询包装器 (LambdaQueryWrapperX)
  - 条件构建优化
  - 分页支持
  - 多数据源支持

#### 3. 缓存框架 (eap-spring-boot-starter-redis)
- **功能**: Redis 缓存统一管理
- **特性**:
  - Redisson 集成
  - 分布式锁
  - 缓存注解
  - 序列化配置

#### 4. 保护机制 (eap-spring-boot-starter-protection)
- **功能**: 系统保护和安全增强
- **特性**:
  - API 签名验证 (@ApiSignature)
  - 幂等性控制 (IdempotentAspect)
  - 防重放攻击
  - 请求限流

#### 5. Web框架 (eap-spring-boot-starter-web)
- **功能**: Web 层统一配置和增强
- **特性**:
  - 统一异常处理
  - 请求响应格式化
  - 跨域配置
  - 参数验证

#### 6. 测试框架 (eap-spring-boot-starter-test)
- **功能**: 单元测试和集成测试支持
- **特性**:
  - 内存数据库测试 (BaseDbUnitTest)
  - Mock 支持
  - 测试数据管理

### 业务增强模块

#### 1. 数据权限 (eap-spring-boot-starter-biz-data-permission)
- **功能**: 细粒度数据权限控制
- **应用场景**: 多租户、部门数据隔离

#### 2. 多租户 (eap-spring-boot-starter-biz-tenant)
- **功能**: SaaS 多租户支持
- **特性**: 租户数据隔离、租户上下文管理

#### 3. IP处理 (eap-spring-boot-starter-biz-ip)
- **功能**: IP 地址相关功能
- **特性**: IP 归属地查询、IP 黑白名单

## 设计模式和架构特点

### 1. 模块化设计
- **Starter 模式**: 采用 Spring Boot Starter 模式，实现自动配置
- **分层架构**: 清晰的分层结构，职责分离
- **插件化**: 各功能模块可独立使用，按需引入

### 2. 统一配置管理
- **依赖版本统一**: 通过 eap-dependencies 统一管理所有依赖版本
- **配置属性**: 使用 @ConfigurationProperties 进行配置管理
- **自动配置**: 利用 Spring Boot 自动配置机制

### 3. 扩展性设计
- **接口抽象**: 核心功能通过接口定义，支持多种实现
- **策略模式**: 如幂等性键解析器支持多种策略
- **AOP 增强**: 使用切面编程实现横切关注点

### 4. 开发效率优化
- **代码生成**: 支持代码自动生成
- **工具类封装**: 提供丰富的工具类库
- **注解驱动**: 大量使用注解简化开发

## 技术亮点

### 1. 查询增强
- **LambdaQueryWrapperX**: 扩展 MyBatis Plus 查询包装器
- **条件构建**: 提供 xxxIfPresent 方法，避免空值查询
- **链式调用**: 支持流畅的链式查询构建

### 2. 安全增强
- **多层安全**: 认证、授权、签名、幂等多重保护
- **灵活配置**: 支持多种安全策略配置
- **Mock 支持**: 开发阶段支持 Mock 模式

### 3. 监控和运维
- **健康检查**: 集成 Spring Boot Actuator
- **指标监控**: 支持多种监控指标
- **日志管理**: 统一日志配置和管理

### 4. 云原生支持
- **微服务**: 可选的微服务框架支持
- **容器化**: 支持 Docker 部署
- **配置中心**: 支持 Nacos 配置中心

## 部署和构建

### Maven 配置
- **多环境**: 支持 dev、dev106、ossrh 等多种部署环境
- **版本管理**: 使用 flatten-maven-plugin 统一版本管理
- **仓库配置**: 支持阿里云、私有仓库等多种 Maven 仓库

### 构建特性
- **注解处理**: 支持 Lombok、MapStruct、Spring Boot Configuration Processor
- **编译优化**: 配置编译参数解决 Spring Boot 3.2 参数发现问题
- **测试集成**: 集成 JUnit 5 测试框架

## 总结

EAP-Common 项目是一个设计良好的企业级基础框架，具有以下特点：

1. **技术先进**: 采用最新的 Spring Boot 3.x 和 JDK 17
2. **架构清晰**: 模块化设计，职责分离明确
3. **功能完整**: 涵盖安全、数据库、缓存、监控等企业应用必需功能
4. **扩展性强**: 支持插件化扩展和自定义配置
5. **开发友好**: 提供丰富的工具类和开发辅助功能
6. **生产就绪**: 具备完整的监控、安全和运维支持

该框架为企业应用开发提供了坚实的技术基础，能够显著提升开发效率和系统质量。
